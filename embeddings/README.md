# Embeddings — Local AI / RAG Setup

This folder stores vector embeddings of every chapter file in the repository.
Combined with `scripts/generate_embeddings.py` and `scripts/semantic_search.py`,
this gives you a fully **local, offline semantic search** engine powered by your GPU.

---

## Hardware Requirements

| Component | Minimum | This Repo's Target |
|-----------|---------|-------------------|
| GPU VRAM | 2 GB | RTX 5070 (12 GB+) |
| RAM | 8 GB | 32 GB recommended |
| Python | 3.10+ | 3.11+ |
| Storage (embeddings) | ~50 MB | ~80 MB for 1,188 chapters |

The `bge-small-en-v1.5` model (default) is 130 MB and runs entirely on CPU if needed,
but will use the GPU automatically when available via CUDA.

---

## Setup

```bash
# 1. Install dependencies
pip install -r scripts/requirements.txt

# 2. Generate embeddings (first run: ~5-10 min on CPU, ~30-60 sec on GPU)
python3 scripts/generate_embeddings.py

# 3. Run a semantic search
python3 scripts/semantic_search.py "sacrificial love that costs the giver everything"
python3 scripts/semantic_search.py "the remnant who hold fast when everyone else falls away"
python3 scripts/semantic_search.py "God dwelling physically among His people"
```

## What Gets Generated

After running `generate_embeddings.py`, this folder will contain:

```
embeddings/
├── README.md               ← this file
├── index.json              ← maps embedding row → chapter file path
├── embeddings.npy          ← float32 numpy array, shape [N_chapters, 384]
├── chronology_gantt.csv    ← (optional) generated by generate_yaml_chronology.py
└── .gitignore              ← excludes .npy from git (too large)
```

The `.npy` file is excluded from Git (see `.gitignore`). Regenerate it locally
by running `generate_embeddings.py`.

## How Semantic Search Works

1. Every chapter's **full text** (key themes + summary + cross-references + keywords) is embedded into a 384-dimensional vector.
2. Your query is embedded into the same space using the same model.
3. Cosine similarity finds the closest chapters — even if they use completely different words.

### Example: What Keyword Search Misses

| Query | Keyword search finds | Semantic search also finds |
|-------|---------------------|---------------------------|
| "sacrificial love" | John 3:16, 1 Cor 13 | Ruth 1 (loyalty unto death), Genesis 22 (father offering son), Hosea (pursuing unfaithful wife) |
| "a remnant that stands" | Isaiah 10, Romans 9 | 1 Kings 19 (7,000 who have not bowed), Revelation 12 (woman in wilderness), Elijah chapters |
| "God dwelling among people" | Exodus 25 (tabernacle), Rev 21 | Genesis 3 (garden walks), Ezekiel 37 (my dwelling among them), John 1 (Word tabernacled) |

## Model Options

Edit the `MODEL_NAME` variable in `generate_embeddings.py` to switch models:

| Model | Size | Quality | Speed |
|-------|------|---------|-------|
| `BAAI/bge-small-en-v1.5` | 130 MB | Good | Fast |
| `BAAI/bge-base-en-v1.5` | 440 MB | Better | Medium |
| `BAAI/bge-large-en-v1.5` | 1.3 GB | Best | Slow (needs 4+ GB VRAM) |
| `sentence-transformers/all-MiniLM-L6-v2` | 90 MB | Good | Fastest |

For an RTX 5070, `bge-large-en-v1.5` will run comfortably and give the best results.

## Integration with Obsidian

If you use Obsidian, the Smart Connections plugin can use your local embeddings
folder directly. Point it to this directory and it will provide semantic similarity
suggestions as you write notes.

Alternatively, use the Text Generator plugin with a local Ollama instance
for full in-Obsidian RAG over this entire biblical knowledge base.
